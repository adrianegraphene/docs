## How It Works: Technical Flow

### Phone Call Flow

<Steps>
  <Step title="Call Attempt">
    Unknown number attempts to call FynCom-protected recipient
  </Step>
  <Step title="Stake Check">
    FynCom middleware intercepts call and checks for stake
    - If no stake: Call blocked, sender notified via SMS/IVR
    - If stake exists: Call proceeds to recipient
  </Step>
  <Step title="Call Duration Tracking">
    System monitors call connection and duration in real-time
  </Step>
  <Step title="Automatic Settlement">
    When call ends:
    - If duration ≥ 25 seconds: Refund stake to sender
    - If duration < 25 seconds: Forfeit stake (80% recipient, 20% treasury)
    - Smart contract executes settlement automatically
  </Step>
</Steps>

### Message Flow (SMS/Email/DM)

<Steps>
  <Step title="Message Sent">
    Unknown sender attempts to message FynCom-protected recipient
  </Step>
  <Step title="Stake Requirement">
    FynCom API intercepts message and requires stake
    - If no stake: Message filtered to spam folder
    - If stake exists: Message delivered to inbox
  </Step>
  <Step title="Thread Monitoring">
    System tracks conversation thread and response count
  </Step>
  <Step title="Automatic Settlement">
    After thread closes or timeout:
    - If ≥ 3 responses in thread: Refund stake to sender
    - If < 3 responses: Forfeit stake (80% recipient, 20% treasury)
    - Smart contract executes settlement automatically
  </Step>
</Steps>

## SDK & API

### Developer Integration

FynCom provides comprehensive SDK and API for platform integrations:

<Tabs>
  <Tab title="JavaScript SDK">
    ```javascript
    import { FynCom } from '@fyncom/sdk';

    const fyncom = new FynCom({
      apiKey: 'your-api-key',
      network: 'mainnet'
    });

    // Require stake for incoming communication
    const result = await fyncom.requireStake({
      sender: 'unknown@example.com',
      recipient: 'user@example.com',
      amount: 0.01,
      refundCriteria: {
        type: 'thread_length',
        minResponses: 3,
        timeoutHours: 72
      }
    });

    // Check stake status
    const status = await fyncom.getStakeStatus(result.stakeId);

    // Manually settle (if automated settlement disabled)
    await fyncom.settleStake(result.stakeId, true); // true = refund
    ```
  </Tab>

  <Tab title="Python SDK">
    ```python
    from fyncom import FynCom

    fyncom = FynCom(
        api_key='your-api-key',
        network='mainnet'
    )

    # Require stake for incoming communication
    result = fyncom.require_stake(
        sender='unknown@example.com',
        recipient='user@example.com',
        amount=0.01,
        refund_criteria={
            'type': 'thread_length',
            'min_responses': 3,
            'timeout_hours': 72
        }
    )

    # Check stake status
    status = fyncom.get_stake_status(result['stake_id'])

    # Manually settle
    fyncom.settle_stake(result['stake_id'], met=True)
    ```
  </Tab>

  <Tab title="REST API">
    ```bash

    curl -X POST https://api.fyncom.com/v1/stakes \
      -H "Authorization: Bearer YOUR_API_KEY" \

      -d '{
        "sender": "unknown@example.com",
        "recipient": "user@example.com",
        "amount": 0.01,
        "refund_criteria": {
          "type": "thread_length",
          "min_responses": 3,
          "timeout_hours": 72
        }

    # Check stake status
    curl https://api.fyncom.com/v1/stakes/{stake_id} \
      -H "Authorization: Bearer YOUR_API_KEY"

    # Settle stake
    curl -X POST https://api.fyncom.com/v1/stakes/{stake_id}/settle \
      -H "Authorization: Bearer YOUR_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{"criteria_met": true}'
    ```
